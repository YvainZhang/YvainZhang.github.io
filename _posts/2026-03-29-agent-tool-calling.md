很多人谈 Agent 时会重点讲“思考”，但工程上真正拉开差距的，经常是工具调用层。

## 1. 为什么工具调用这么关键

因为没有工具，Agent 大多只能停留在文本层。  
一旦接上工具，它才可能：

- 读文件
- 查网页
- 调 API
- 跑代码
- 写回结果

## 2. 一个好工具接口应该长什么样

至少要做到：

- 输入清晰
- 输出结构稳定
- 错误可捕获
- 权限边界明确

如果工具设计本身不稳定，模型只会被迫在噪声里工作。

## 3. 常见风险

- 工具过多，模型乱选
- 返回格式不统一
- 错误处理不完整
- 权限过大

这些问题不解决，Agent 很容易从“会做事”变成“会制造混乱”。
