第一次做 Agent，最容易犯的错就是一上来就堆框架、堆插件、堆工具。  
更稳妥的做法，是先做一个最小可运行版本。

## 1. 最小 Agent 只需要什么

你真正需要的通常只有四个部分：

- 输入解析
- 一个或两个工具
- 简单状态保存
- 最终结果输出

比如天气助手、网页摘要器、文件整理器，这类任务都很适合当第一步。

## 2. 为什么要先做最小闭环

因为它能帮你尽快看清楚这三个核心问题：

- 模型到底负责什么
- 工具接口长什么样
- 中间状态该怎么保存

这比先学一大堆框架概念更有用。

## 3. 第一个 Agent 应该避免什么

- 不要一开始就接太多工具
- 不要让任务目标太模糊
- 不要一上来就做多代理协作

先把单 Agent 单任务跑顺，比什么都重要。

## 4. 一个合理的起点

如果要选第一个题目，我会优先建议下面几类：

- 天气和建议生成
- 文档读取与总结
- 指定目录的文件整理
- 简单问答加工具调用

这些任务足够小，但已经能覆盖 Agent 最核心的机制。
