Agent 的大脑通常是大语言模型，但 Agent 不是把模型直接塞进流程里就结束了。  
要把系统做好，先得知道 LLM 擅长什么，不擅长什么。

## 1. LLM 擅长的事情

大模型最适合做的是：

- 理解自然语言目标
- 根据上下文做推理
- 生成结构化或半结构化输出
- 在已有信息上做总结和改写

这使它非常适合担任控制中心。

## 2. LLM 不擅长的事情

大模型天然不稳定的部分也很明确：

- 容易幻觉
- 对外部世界没有实时事实保证
- 多步任务中容易偏航
- 长流程里状态容易丢失

所以工程上不能把“模型会想”误当成“系统会做”。

## 3. 面向 Agent 时最关键的三个问题

### 能不能稳定决定下一步

Agent 要求模型输出的不只是答案，而是动作决策。

### 能不能和工具接口协同

模型要学会在合适时机调用合适工具，而不是一路空想。

### 能不能在长任务里维持上下文

一旦任务拉长，记忆设计和上下文管理就会变成系统问题，而不只是模型问题。

## 4. 结论

LLM 是 Agent 的核心，但不是全部。  
真正可靠的 Agent，永远是“模型能力 + 系统约束 + 工具接口 + 状态管理”的组合结果。
