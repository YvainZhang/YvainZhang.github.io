理解 Agent，最重要的一步不是背框架名，而是先把它内部最常见的几个组件拆开。

## 1. LLM：负责理解和推理

LLM 是整个系统的推理核心。它主要做几件事：

- 理解用户目标
- 判断当前任务状态
- 生成下一步动作建议
- 总结结果并生成输出

但它本身并不会天然执行外部动作，所以它只是“大脑”，不是完整系统。

## 2. Planning：负责拆任务

规划模块的作用，是把模糊目标变成可执行步骤。

比如“帮我整理一份 AI Agent 学习路线”，中间可能会拆出：

1. 收集资料
2. 分组归类
3. 判断优先级
4. 输出学习顺序

如果没有 Planning，模型就容易停留在“会说”，但不会稳定推进。

## 3. Tool use：负责落地动作

工具层决定了 Agent 能做什么。常见工具包括：

- 搜索
- 代码执行
- 文件读写
- API 调用
- 数据库查询

Agent 一旦接上工具，就不再只是文本系统，而开始具备执行能力。

## 4. Memory：负责保留状态

记忆至少有两层：

- 短期记忆：当前任务上下文
- 长期记忆：用户偏好、知识片段、历史结果

没有 Memory，Agent 每轮都像第一次见面，无法形成连续工作流。

## 5. 组件之间真正重要的关系

单独看每个组件都不复杂，真正难的是协同：

- LLM 决定下一步
- Planning 保证任务不乱跳
- Tool use 执行动作
- Memory 保存状态和反馈

这四个部分闭环以后，Agent 才像一个系统，而不是一段 prompt。
