“问答”这个词很容易让人以为 Agent 只是把聊天做复杂了。  
但一个真正的 Agent 问答系统，往往不是直接生成答案，而是先做若干中间动作。

## 1. Agent 问答和普通问答有什么不同

普通问答：

- 输入问题
- 直接输出答案

Agent 问答：

- 判断问题类型
- 检索或调用工具
- 汇总中间结果
- 返回最终答案

## 2. 什么时候值得这么做

当问题需要：

- 外部知识检索
- 多步推理
- 状态延续
- 结果验证

这时问答就不再是简单生成，而更像一段任务流。

## 3. 更务实的理解

问答只是外壳。  
真正重要的是系统内部有没有：

- 工具调用
- 状态管理
- 结果筛选
- 错误处理

这些机制在，Agent 才有工程意义。
