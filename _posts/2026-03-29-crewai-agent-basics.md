CrewAI 这类框架最显眼的特点，是它不只关心“一个 Agent 怎么做事”，而是强调多个角色如何协同。

## 1. 它解决的核心问题

在很多复杂任务里，单一 Agent 往往会变得又长又混乱。  
CrewAI 试图把任务拆到不同角色上，例如：

- 研究者
- 写作者
- 审核者

这样做的目的，是让职责边界更清楚。

## 2. 适合什么任务

它更适合下面这种场景：

- 任务本身可明确拆分
- 不同子任务之间职责清楚
- 最终结果需要汇总或审阅

如果只是简单单步任务，上多角色往往没必要。

## 3. 使用时最容易忽视的点

多 Agent 协作不等于自动变强。  
如果角色边界不清、共享上下文混乱、目标不明确，多代理只会让系统更难调试。

所以它更像结构化组织工具，而不是性能加成器。
