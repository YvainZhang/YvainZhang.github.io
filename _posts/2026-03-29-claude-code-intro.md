Claude Code 这类工具真正重要的地方，不是“它会写代码”，而是它把模型放到了一个可执行、可读写、可迭代的工程环境里。

## 1. 它和普通网页聊天的差别

终端或代码环境中的 Agent，能够直接接触：

- 代码库
- 文件系统
- 命令行工具
- 测试与构建输出

这意味着它可以不止解释问题，而是直接参与修改和验证。

## 2. 适合什么场景

- 看一个陌生仓库
- 改一个小功能
- 跑测试并修问题
- 批量重构

这些都是 Agent 编码工具最容易体现价值的地方。

## 3. 使用时最重要的原则

不要把它当自动写代码机器，而要把它当一个具备上下文和执行能力的协作工具。  
关键不只是生成代码，而是：

- 有没有正确理解仓库
- 有没有验证修改结果
- 有没有控制修改范围
