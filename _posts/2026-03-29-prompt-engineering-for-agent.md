在 Agent 系统里，提示词的意义比普通问答更大，因为它直接影响：

- 任务理解方式
- 工具调用策略
- 输出格式稳定性
- 失败时的恢复逻辑

## 1. 提示词最重要的作用不是“写得花”

真正有效的提示词，通常不是辞藻更丰富，而是边界更清楚。

至少要把下面这些说清楚：

- 目标是什么
- 允许使用什么工具
- 结果需要什么格式
- 遇到失败时怎么处理

## 2. Agent 场景里常见的提示词结构

一个比较实用的结构通常包括：

- 角色定义
- 任务目标
- 可用工具
- 执行约束
- 输出格式

如果任务比较复杂，再补充：

- 终止条件
- 错误处理策略
- 优先级

## 3. 为什么提示词工程不能替代系统设计

很多问题看上去像 prompt 不够好，实际上是系统接口没设计好：

- 工具返回不稳定
- 中间状态不可见
- 错误没有重试机制

提示词可以帮助模型更听话，但不能替代工程约束。

## 4. 更合理的理解

在 Agent 里，提示词更像控制接口。  
它负责告诉模型：

- 你是谁
- 你现在要干什么
- 你能做什么
- 你必须怎样返回

提示词写得越结构化，系统越容易稳定。
